* Packages

Package declarations go here. Use =package!= to declare packages, then run
=doom sync= to install them.

#+begin_src elisp :tangle packages.el

* Configuration

This section is loaded after all modules have loaded. Use =after!= or
=use-package!= to configure packages.

Some functionality uses this to identify you, e.g. GPG configuration, email
clients, file templates and snippets. It is optional.

#+begin_src elisp :tangle config.el
;;; config.el -*- lexical-binding: t; -*-

;; (setq user-full-name "John Doe"
;;       user-mail-address "john@doe.com")
#+end_src

** Appearance

Doom exposes five (optional) variables for controlling fonts:

- =doom-font= — the primary font
- =doom-variable-pitch-font= — a non-monospace font (where applicable)
- =doom-big-font= — used for =doom-big-font-mode= (presentations, streaming)
- =doom-symbol-font= — for symbols
- =doom-serif-font= — for the =fixed-pitch-serif= face

#+begin_src elisp :tangle config.el
(setq doom-theme 'doom-one
      doom-font (font-spec :family "Hack Nerd Font" :size 12)
      doom-variable-pitch-font (font-spec :family "Hack Nerd Font")
      doom-big-font (font-spec :family "Hack Nerd Font" :size 18))
#+end_src

** Line settings

If set to =nil=, line numbers are disabled. For relative line numbers, set
this to ='relative=.

#+begin_src elisp :tangle config.el
(setq display-line-numbers-type 'relative)
#+end_src

** Org directory

If you use =org= and don't want your org files in the default location,
change =org-directory=. It must be set before org loads.

#+begin_src elisp :tangle config.el
(setq org-directory (or (getenv "XDG_DATA_HOME")
                        (concat (getenv "HOME") "/.local/share"))
      org-roam-directory (expand-file-name "org-roam/" org-directory))
#+end_src

** Misc

(setq undo-limit 80000000         ; 80MB (raise undo limit)
      evil-want-fine-undo t       ; more granular undo
      auto-save-default t)        ; auto-save files
#+end_src

** Tips

Whenever you reconfigure a package, make sure to wrap your config in a
=with-eval-after-load= block, otherwise Doom's defaults may override your
settings. The exceptions to this rule:

- Setting file/directory variables (like =org-directory=)
- Setting variables which explicitly tell you to set them before their
  package is loaded
- Setting doom variables (which start with =doom-= or =+=)

Useful functions/macros for configuring Doom:

- =load!= for loading external =*.el= files relative to this one
- =add-load-path!= for adding directories to the =load-path=
- =map!= for binding new keys
